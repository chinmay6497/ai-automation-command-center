Webhook contract — n8n · Zapier · Make · anything

Point any HTTP node at these endpoints. Responses are 202 Accepted with an item_id; processing is asynchronous.

Submit a lead

curl -X POST https://YOUR-APP.onrender.com/api/leads \
  -H "Content-Type: application/json" \
  -H "X-API-Key: YOUR_INTAKE_KEY" \
  -d '{
    "name": "Jane Doe",
    "email": "jane@acme.com",
    "company": "Acme Inc",
    "message": "We need invoice automation, budget approved, urgent."
  }'

Submit an inbound email

curl -X POST https://YOUR-APP.onrender.com/api/emails \
  -H "Content-Type: application/json" \
  -H "X-API-Key: YOUR_INTAKE_KEY" \
  -d '{
    "name": "Tom Becker",
    "email": "tom@beckerco.com",
    "subject": "Integration question",
    "body": "Does your platform integrate with HubSpot?"
  }'

Authentication: if INTAKE_API_KEY is set on the server, machine callers must send X-API-Key. The built-in dashboard and contact page remain open (same-origin).

Poll results: GET /api/items/{item_id} returns status, score, priority, draft, risk, and the full agent trace.

Typical n8n recipe

  1. Trigger node — Gmail / LinkedIn / Typeform / Twilio / CRM watch
  2. Set node — map fields to the contract above (name, email, message)
  3. HTTP Request node — POST to /api/leads with the X-API-Key header
  4. Wait + HTTP node (optional) — poll /api/items/{id} and branch on status: route approved drafts to your sender of choice (Outlook, LinkedIn DM, WhatsApp…)

The Command Center owns judgment and governance; n8n owns connectors and delivery. Use each for what it's best at.

Enable channels (each is one env var)

✈️ Telegram (2 min)

  1. Message @BotFather, send /newbot
  2. Copy the token into TELEGRAM_BOT_TOKEN
  3. Deployed? Set BASE_URL (webhook mode). Localhost? Leave empty (polling mode)

📧 Resend (5 min)

  1. Free account at resend.com → API key into RESEND_API_KEY
  2. Sandbox sender works for your own inbox; verify a domain (free) to email anyone, then set EMAIL_FROM

💬 Slack (2 min)

  1. Create an Incoming Webhook in your Slack workspace
  2. Paste the URL into SLACK_WEBHOOK_URL
